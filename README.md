# NYC Taxi Prediction Pipeline

This repository contains production-ready Machine Learning pipelines for the **NYC Green Taxi dataset**. It addresses two distinct tasks:

1.  **Regression:** Predicting the tip amount ($).
2.  **Classification:** Predicting the payment type (Credit, Cash, Other).

The project emphasizes robust evaluation using **Cross-Validation** for hyperparameter tuning and **Statistical Hypothesis Testing** for model comparison.

## 🛠️ Regression Task (Tip Amount)
**Location:** `src/regression/`

Uses **XGBoost Regressor** to predict the continuous tip value.

NOTE: Due to practicity reasons, only one model was chosen (Practice focus)

### Workflow
1.  **Configure (`config.py`):** Define the experiment tag (e.g., `xgb_3feats`) and feature set.
2.  **Train (`run.py`):**
    *   Loads data and creates a hold-out Test Set (fixed seed).
    *   Runs **5-Fold Cross-Validation** to determine the average optimal number of trees (Early Stopping).
    *   Retrains on the full training set using the calculated average epochs.
    *   Evaluates on the Test Set and exports a `.pkl` report.
3.  **Compare (`tests.py`):**
    *   Loads all reports from `runs/regression/`.
    *   Ranks models by **MAE** (Mean Absolute Error).
    *   Performs the **Wilcoxon Signed-Rank Test** to check if the Champion model is statistically superior to Challengers (robust to outliers).

### Usage

```bash
cd src/regression
python run.py   # Run experiment
python tests.py # Compare results (whithin 2 or more runs!)
```

## 📊 Classification Task (Payment Type)
**Location:** `src/classification/`

Uses **XGBoost Classifier** to predict payment methods.
*   **Target Mapping:** `0`=Credit, `1`=Cash, `2`=Other (Merged 3 and 4).

### Workflow
1.  **Configure (`config.py`):** Define experiment tag and features.
2.  **Train (`run.py`):**
    *   Performs **Stratified Train/Test split** to handle class imbalance. NOTE: No rebalancing strategy was used for training.
    *   Runs **5-Fold Cross-Validation** monitoring **Weighted F1-Score**.
    *   Retrains on the full training set using the average optimal epochs.
    *   Exports a report containing the Confusion Matrix and F1 scores.
3.  **Compare (`tests.py`):**
    *   Loads all reports from `runs/classification/`.
    *   Ranks models by **Weighted F1-Score**.
    *   Performs **McNemar's Test** to statistically analyze the contingency table of disagreements between models.

### Usage

```bash
cd src/classification
python run.py   # Run experiment
python tests.py # Compare results
```

## 📈 Methodology

*   **Cross-Validation:** Used strictly for determining the optimal training duration (epochs) to prevent overfitting, not for final evaluation.
*   **Final Evaluation:** All metrics are reported on a strictly held-out **Test Set** that remains constant across all experiments (guaranteed by `RANDOM_SEED`).
*   **Statistical Rigor:** We do not rely solely on metric improvement. We use statistical tests (**Wilcoxon** for regression, **McNemar** for classification) to differentiate between random noise and actual model improvement.
